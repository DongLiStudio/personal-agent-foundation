# Obsidian 结构确认与连接

本流程只在用户配置真实 Obsidian Vault 时执行。目标是让 `GLOBAL/OBSIDIAN_LINK.md` 保持与模板同构，只替换“目录理解”和“优先阅读”中涉及用户 Vault 实际结构的部分。它不是安装日志，不记录 Vault 原始绝对路径；真实路径只用于本轮创建和验证 `GLOBAL/obsidian-resource`。

## 一、用通俗方式确认，不问抽象术语

不能只询问软连接路径，也不要直接问“目录理解、优先阅读入口是什么”这类抽象问题。先用普通话说明：

```text
我需要先理解你的 Obsidian 资料大概怎么分区，这样以后 Agent 才知道该先看哪里、哪些地方不能碰。
我可以先只看 Vault 根目录第一层文件夹，初步猜一下每个目录是干什么的，然后列给你确认。
如果我猜得不对，你直接告诉我改哪里就行。
```

然后确认：

1. 这是已有 Vault、空 Vault，还是准备新建的 Vault。
2. 是否允许只读列出 Vault 根目录第一层文件和文件夹。
3. 是否已有总览、首页、README、MOC、索引、仪表盘或导航文件。
4. 哪些目录默认不该碰，例如日记、附件、隐私、财务、证件、家庭资料或 `.obsidian`。
5. 是否允许 Agent 只读查看用户点名的索引文件或目录第一层，用来确认初步判断。

如果用户直接给出清楚结构，不必额外扫描。否则默认建议做有界只读发现来辅助判断；用户拒绝发现时，再请用户用自己的话描述结构。只有用户明确说“之后再定”“暂不连接”或“当前没有 Vault”，才把 Obsidian 标记为未配置或待配置。

## 二、必要时做有界发现

只有用户同意后才能查看现有 Vault。发现范围默认限于：

- 列出 Vault 根目录一层的文件和目录。
- 查看用户点名的索引文件或目录一层。
- 确认用户明确指定路径是否存在。

不得递归遍历整个 Vault，不得进入 `.obsidian`，不得批量读取正文，不得跟随 Vault 内链接到其他位置。附件、私密日记、财务、身份资料等内容只有在用户明确点名并授权后才能读取。

发现后先由 Agent 自己做初步解析，再列给用户确认，例如：

```text
我先这样理解你的目录：
- A：像是事实资料/业务信息。
- B：像是项目资料。
- C：像是附件或功能性文件，默认不遍历。

你看哪里不对？需要我把哪个目录理解成别的用途？
```

文件夹名称不能单独证明其用途。发现结果与用户描述不一致时，应展示差异并继续询问，不要自行选择。

## 三、生成用户专属连接约定

基于已确认信息，拟定已安装实例中的 `GLOBAL/OBSIDIAN_LINK.md`。必须保留模板的整体结构：

- 标题 `Obsidian Vault Link`。
- 个人 Obsidian 库入口使用安装后 `AGENT_ROOT` 下的 `GLOBAL/obsidian-resource`，不写 Vault 原始绝对路径。
- `定位` 与 `安全规则` 保持通用骨架。
- `目录理解` 用用户真实目录名称替换模板中的待确认项；不适用项明确写“不适用”。
- `优先阅读` 用用户确认的总纲、索引、重点项目、方法论或知识路由入口替换待确认项。
- `未配置判定` 保留，用于说明哪些问题仍待确认。

先把完整草案展示给用户。只有用户确认后，才替换已安装实例中的通用内容。不要修改产品仓库内的模板源文件。不得生成过度泛化的表格来替代上述结构。

## 四、创建与验收链接

目录映射确认后才创建链接：

- Windows 优先创建 Junction。
- macOS/Linux 创建 symlink。

创建前确认 Vault 真实存在，且不是 Agent 根目录、GLOBAL、它们的父目录或内部目录。创建后独立回读：

- 链接类型。
- `GLOBAL/obsidian-resource` 的入口路径。
- 解析后的 Vault 目标。
- `GLOBAL/OBSIDIAN_LINK.md` 的最终内容。

不得把 Vault 复制进 GLOBAL，也不得把创建链接等同于获得写权限。

## 五、特殊情况

- 未配置 Vault：不创建链接，保留通用说明并明确标记未配置。
- 空 Vault 或目录很少、尚未形成结构：先推荐默认五类结构（信息、资源、想法、知识、特殊），用通俗说明每类用途，并询问是否采用。用户同意后，先展示将创建的目录和入口文件清单，取得单独确认后再创建；用户不同意时，询问他希望改成什么结构；用户仍跳过时，才把“目录理解”和“优先阅读”标记为待补充。
- 用户采用作者默认结构：仍展示识别出的映射并确认，不能仅凭同名目录自动接受。
- 用户暂时说不清结构：先提供“你可以描述目录用途，或允许我只读列出根目录一层来辅助判断”的选择；用户仍明确表示之后再定时，完成其他安装步骤，把 Obsidian 标记为待配置；不要用猜测内容完成验收。
