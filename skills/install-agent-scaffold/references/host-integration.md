# 宿主 Agent 集成

## 图形交互

图形提问、目录选择和确认由当前 Agent 宿主提供。宿主有原生交互工具时使用它；没有时退回普通对话。不要随产品发布自制 GUI。

任何宿主支持可视化安装面板、图形提问、目录选择、选项卡片或交互式呈现能力时，安装器必须先尝试用它展示安装选择和确认摘要。当前宿主可用时必须优先尝试可视化安装面板或确认摘要。若当前会话明确可用可视化能力，却未触发可视化安装面板或确认摘要，本次安装不得直接继续收集配置；必须先说明阻断原因并退回普通对话确认。可视化至少覆盖：

宿主能力名称以当前环境为准。例如 Codex 中可能表现为 `Visualize` 插件/能力；其他 Agent 可能表现为表单、卡片、目录选择器、Webview 或安装向导。安装器应识别并使用当前宿主提供的能力，不要把某个宿主或插件名称当作唯一实现。

- 推荐安装路径、权限影响和目标目录状态。
- 初始安装只需要的最小配置：Agent 根目录、时区和通用助手项目名。
- dry-run 后的写入计划、风险提示和待确认操作。
- GLOBAL 与 Skills 恢复后的连接选择：飞书、GitHub、Obsidian 是否现在连接。

可视化能力只用于帮助用户理解和选择；文件写入仍必须由 `scaffold_guard.py` 的 audit、plan、install 和 verify 门禁完成。当前宿主没有可视化能力、插件不可用、无法嵌入交互或可视化生成失败时，必须退回普通对话逐项确认，并在结果中说明“图形交互未使用”的具体原因。

## 全局 Skill

- Codex：优先使用当前 Skill 安装器；只有明确解析到用户级全局 Skill 目录时才直接同步文件。
- 其他 Agent：读取该宿主当前官方 Skill/扩展说明或设置界面，不根据其他宿主路径猜测。
- 宿主不能安装 Skills 时，保留 GLOBAL 源稿，报告“源稿已安装、宿主尚未启用”，并给出人工入口。

## 全局个性化提示词

安装器不替宿主修改未确认的设置文件。引导用户在当前 Agent 设置中查找类似入口：

- Custom instructions
- Personalization
- Global instructions
- User rules
- System prompt / profile prompt

说明：GLOBAL 是磁盘治理基座；全局个性化提示词是宿主层偏好，两者应同时配置但不能互相替代。

向用户展示并要求保存以下完整内容，不要自行压缩或改写：

```text
开始任何项目工作前，必须完整读取当前项目根目录的 AGENTS.md、README.md 和 STATUS.md，并遵循 AGENTS.md 声明的上层治理、持续上下文、项目规则及按需披露路径。

其中：
- AGENTS.md 定义工作规则、授权边界、身份路由和上层治理关系；
- README.md 定义项目定位、资料入口和长期结构；
- STATUS.md 定义当前阶段、最新事实、阻塞项和下一步。

用户当前明确指令具有最高优先级；项目实时文件优先于历史对话、摘要和旧记录。不得因当前请求较简单而跳过上述入口，也不得把历史上下文当作当前项目事实。

发生会话压缩、任务迁移、跨任务委派、长期岗位接单或项目归属变化后，必须重新读取并校准这三个文件。准备修改文件、调用项目身份、操作外部系统或提交 Git 前，如上下文可能已经变化，应再次核对相关规则和 STATUS。

委派任务时，必须要求接收方执行同样的项目上下文读取。接收方遇到需要确认的实质问题、权限阻塞或任务完成时，必须主动回报来源任务；未回报不得视为完成。已有明确决定不得重复询问。
```

如果宿主对提示词有长度限制，先明确告诉用户限制和无法完整保存的部分，再协商缩减版本；不得静默删减。

用户确认保存后，要求其说明已保存到哪个宿主设置入口；宿主支持读取设置时回读核对完整文本，无法回读时明确标记为用户确认，而不是程序验证。
