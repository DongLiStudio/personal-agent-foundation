# Obsidian Vault 连接约定

## 当前连接

- Vault：`{{OBSIDIAN_VAULT_PATH}}`
- Agent 入口：`{{AGENT_ROOT}}/GLOBAL/obsidian-resource`
- 状态：安装时确认；未配置 Vault 时不创建入口。

## 定位

Obsidian 是用户自己的外部知识库，不是 Agent 项目工作区。Agent 可以在用户授权范围内选择性读取长期资料，但不得假设 Vault 采用某套固定目录结构。

## 已确认的目录映射

安装时应根据用户的实际 Vault 结构填写本节。未与用户确认前保持“待确认”，不得套用作者或其他用户的目录名称。

| 用途 | 用户的目录或文件 | Agent 使用方式 |
| --- | --- | --- |
| 总览入口 | 待确认 | 选择性读取 |
| 长期资料 | 待确认 | 选择性读取 |
| 项目资料 | 待确认 | 选择性读取 |
| 日记或临时记录 | 待确认 | 默认不读取 |
| 附件与媒体 | 待确认 | 默认不遍历 |

## 优先阅读入口

- 待确认。

按任务需要选择性读取，不要默认全量扫描。递归审计、迁移、索引扫描和批量盘点默认不得进入 `obsidian-resource`。

## 访问边界

- 默认权限：只读。
- 允许写入的目录：无；如用户授权，应在此明确记录。
- 明确排除的目录：`.obsidian`；其他排除项由用户确认后补充。
- 未解决的结构问题：待确认。

未经用户明确授权，不要创建、修改、删除、移动或重命名 Vault 中的文件，不要批量格式化或修改 `.obsidian` 配置，也不要对 `obsidian-resource` 做递归移动、复制或删除。

如果需要沉淀新内容，优先在对应 Agent 项目工作区生成草稿，再由用户确认是否写入 Vault。
