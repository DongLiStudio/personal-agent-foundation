# Obsidian Vault Link

个人 Obsidian 库入口：

`{{AGENT_ROOT}}\GLOBAL\obsidian-resource`

## 定位

Obsidian 是外部知识库，不是 Agent 项目工作区。

Agent 可以读取它来理解长期背景、业务资料、人物资源、想法和方法论，但不应把当前任务直接写回 Obsidian，除非用户明确要求。

## 安全规则

默认只读。

未经用户明确授权，不要在 Obsidian 库中执行以下操作：

- 创建文件
- 修改文件
- 删除文件
- 移动或重命名文件
- 批量格式化
- 改动 `.obsidian` 配置
- 对 `obsidian-resource` 链接目录做递归移动、复制或删除

如果需要沉淀新内容，优先在对应 Agent 项目工作区生成草稿，再由用户确认是否写入 vault。

## 目录理解

- `信息`：事实信息、业务详情、合作公司、通用信息、流程、政策、公司资料。
- `资源`：人脉、基础设施、物资、项目资源、自媒体资源。
- `想法`：发展规划、项目创意、组织思想、业务/工具/游戏类构想。
- `知识`：方法论、AI、前端、后端、游戏开发、股票等技术和认知沉淀。
- `特殊`：模板、附件、字体、待办、Obsidian 功能性支持文件。

## 优先阅读

按任务需要选择性阅读，不要默认全量扫描。

递归审计、迁移、索引扫描和批量盘点默认不得进入 `obsidian-resource`。确实需要长期资料时，先阅读本文件规则，再选择性读取具体文件；禁止对 Obsidian 库做全量遍历。

- `README-总纲.md`
- `信息\README-信息.md`
- `资源\README-资源.md`
- `想法\README-想法.md`
- `特殊\README-特殊.md`
- `想法\发展\优先要做的重点项目.md`
- `知识\方法论\方法论路由：方法论的方法论.md`
