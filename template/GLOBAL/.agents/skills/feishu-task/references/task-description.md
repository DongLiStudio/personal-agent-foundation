# 任务标题、详细描述与验收规则

## 标题

- 使用动宾结构，简短说明最终交付，例如“制作横版与竖版品牌壁纸”。
- 避免“处理一下”“跟进事情”“看一下”等空泛标题。
- 多个紧密相关且共享截止时间、负责人和目标的交付可合并，并在描述中拆分。
- 目标、负责人、截止时间或验收边界明显不同的交付应拆成多个任务。

## 描述结构

按上下文选用以下章节，删除空章节：

1. `【任务目标】`：最终需要达成的业务结果。
2. `【背景与已确认结论】`：任务来源、已认可版本、最终反馈和被替换的旧决定。
3. `【交付内容】`：成品、文档、链接、格式、数量和版本。
4. `【具体执行要求】`：步骤、层级、位置、尺寸、字段、文案、路径、协作分工等。
5. `【禁止项与边界】`：不得改动的内容、合规限制、品牌禁用项和范围边界。
6. `【时间节点】`：开始、阶段检查、评审、提交和最终截止；标明哪些已写入任务 start/due，哪些只是描述中的中间节点。
7. `【负责人和关注人】`：姓名、角色与具体分工。
8. `【验收标准】`：可逐项核对的完成条件。
9. `【来源与推定】`：截图、文件、消息、链接和重要推断。

## 精确信息

- 原样保留用户给出的正式文案、数字、名称、尺寸、文件格式、路径和链接。
- 不擅自润色必须逐字一致的文案；确需纠错时先说明。
- 对“这个版本可以”“不要黑底”“文案改成……”等连续反馈，整理成保留项、修改项和禁用项。
- 后续更正覆盖旧方案；旧方案只在解释背景确有必要时保留，并标明已废弃。
- 无法确认的人名、金额、日期、产品名或职责标为待确认，不得补写猜测。

## 视觉与内容任务

写清以下适用项：

- 已认可的参考版本和必须保留的视觉特征。
- 构图层级、相对位置、颜色、材质、风格、字体和留白。
- 横竖比例、像素尺寸、桌面/移动端适配和安全区。
- 精确文案、主副标题层级、标点、换行和文字顺序。
- 输出格式、透明背景要求、文件命名、版本数量和源文件。
- 禁用效果、不得替换的 Logo/素材及品牌限制。
- 逐字、逐尺寸、逐文件的验收方法。

## 会议与项目任务

写清以下适用项：

- 背景、目标、已决定事项和仍待确认事项。
- 行动项、负责人、协作者、依赖关系和前置条件。
- 阶段节点、开始时间、评审机制、最终交付、截止时间和风险。
- 数据口径、系统字段、业务范围和合规边界。
- 可观察、可复核的验收结果，不使用“效果良好”等空泛表述。

## 来源与推定

- 标明关键要求来自哪条消息、截图、文件、会议或链接。
- 只记录影响执行的重要来源，不复制无关聊天全文。
- 可合理推断但没有明示的信息标记“根据上下文推定”。
- 推断若会改变负责人、截止日期、交付范围或验收结果，应先询问，不直接写入。
