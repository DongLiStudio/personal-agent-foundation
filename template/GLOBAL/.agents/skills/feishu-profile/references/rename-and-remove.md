# 重命名与安全删除

## 重命名

1. 实时列出 Profile，确认源名称唯一且目标名称不存在。
2. 使用 `rg` 在当前项目、GLOBAL 映射和用户明确纳入范围的其他项目中搜索精确 Profile 名。
3. 列出影响：默认身份、客户路由、业务 Skill、脚本、任务文档和历史记录。
4. 用户要求重命名即授权修改 CLI 名称和当前项目引用；跨项目修改仍须受各项目边界约束。
5. 执行 CLI rename 后回读 Profile 列表、`auth status --verify` 和 `whoami`。
6. 更新当前项目规则和非历史引用；历史事实保留旧名称并注明时间，不盲目替换。

重命名只改变本地 Profile 逻辑名称，不假设会修改飞书后台应用名、企业名或其他电脑配置。

## 删除前审计

删除属于破坏性本地认证操作，必须明确确认。执行前：

- 回读 Profile、应用和用户身份，避免同名误删；
- 搜索当前项目与 GLOBAL 的默认/例外引用；
- 确认是否仍有未完成任务、自动化或客户资源依赖；
- 说明删除的是本机/当前 workspace Profile，还是另有飞书后台应用处置需求；
- 不把“删除本地 Profile”解释为“删除企业自建应用”。

## 删除顺序

1. 用户明确确认目标 Profile 和删除范围。
2. 先修改当前授权项目中的活跃路由，不能留下指向不存在 Profile 的当前规则。
3. 执行 `auth logout` 或 `profile remove` 前读取当前 CLI `--help` 并遵循确认参数。
4. 删除后回读 Profile 列表，确认其他主体未受影响。
5. 报告仍存在的项目引用、其他电脑配置和飞书后台应用；不代替用户扩大删除范围。

## 禁止事项

- 不自动删除 active/default Profile。
- 不因 token 失效自动删除 Profile；应先重新授权。
- 不批量清空 CLI 配置解决单个账号问题。
- 不删除飞书后台企业应用，除非用户单独明确要求且已完成影响确认。
- 不删除或改写历史记录来掩盖曾经使用过的 Profile。
